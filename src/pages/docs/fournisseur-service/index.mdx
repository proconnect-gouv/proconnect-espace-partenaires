# Documentation Fournisseur de Service

Vous souhaitez implémenter ProConnect sur votre site ? Vous êtes au bon endroit ! Cette documentation présente l'ensemble des informations à connaître.

## 1. 👩‍🏫 Préambule

Cette documentation est à destination des Fournisseurs de Services souhaitant intégrer ProConnect. ProConnect est implémenté sur deux plateformes, une dite "RIE" (l’agent se connecte depuis le RIE, à un FS RIE via un FI RIE) et une autre dite "Internet" (l'agent se connecte depuis Internet, à un FS Internet via un FI Internet). Toutefois il est possible d'utiliser "l'hybridge Internet/RIE", qui permet aux agents, depuis un Fournisseur de Service **sur Internet**, de s'identifier via un Fournisseur d'Identité **sur le RIE**.

Avant toute chose, vérifiez votre [éligibilité à intégrer le bouton ProConnect sur vos services](./eligibilite_installation.md).

## 2. ⚙️ Les étapes pour intégrer ProConnect

<VerticalStepper>
  <Step number={1} title="Familiarisez-vous avec les concepts de base">
    Je consulte les [concepts de base](../ressources/flux_oidc.md) du protocole OpenID Connect (OIDC).

    NB: si vous êtes Fournisseur de Service, ProConnect est votre _provider_ et vous êtes _client_.

  </Step>
  <Step number={2} title="Définissez votre parcours utilisateur">
    Je définis mon parcours utilisateur pour la connexion : voir [nos
    recommandations](./recommandation_parcours.md)
  </Step>
  <Step number={3} title="Lancez les développements en test">
    Je [crée une application OIDC](/apps) directement sur ce portail. Elle sera associée à mon adresse email. J'obtiens immédiatement un `client_id` et un `client_secret` de test.
  </Step>
  <Step number={4} title="Implémentez la cinématique OIDC">
    J’ai implémenté la cinématique OIDC (Authorization Code Flow): voir
    l'[implémentation technique](./implementation_technique.md)
  </Step>
  <Step number={5} title="Contractualisez avec la DINUM">
    Je contractualise officiellement ma collaboration avec la DINUM en remplissant
    le [DataPass dédié](./datapass-fs.md). Je peux passer à l'étape suivante même
    si le Datapass n'est pas encore complété ou approuvé pour gagner du temps.
    Après complétude, le Datapass me revient signé dans les **2 jours ouvrés**.
  </Step>
  <Step number={6} title="Récupérez vos identifiants de production">
    Je remplis [le formulaire
    dédié](https://www.demarches-simplifiees.fr/commencer/demande-creation-fs-fca)
    avec les informations de production. Après complétude, je reçois mon
    `client_id` et mon `client_secret` de production dans les **2 jours ouvrés**.
  </Step>
  <Step number={7} title="Ouvrez votre service">
    Ouverture du service en production 🚀
  </Step>
  <Step number={8} title="Référencez-vous dans l'annuaire des services">
    Je signale à l'équipe ProConnect que je souhaite apparaître sur [sa liste de services connectés](https://www.proconnect.gouv.fr/services). Si la description du service rendue a bien été remplie dans le Démarches Simplifiées (avec verbe à l'infinitif au début de la description), l'équipe mettra à jour le site.
  </Step>
</VerticalStepper>
